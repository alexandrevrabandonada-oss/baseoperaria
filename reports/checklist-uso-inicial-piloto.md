# Checklist de Uso Inicial do Piloto

Data: 15/04/2026

## Antes de liberar entrada

- [ ] Empresa Piloto criada
- [ ] Unidade Principal criada
- [ ] Setores Producao e Expedicao criados
- [ ] Turnos Dia e Noite criados
- [ ] Categorias de condicoes ativas
- [ ] Categorias economicas ativas
- [ ] Buckets de anexos criados
- [ ] Admin inicial vinculado como `owner` ou `admin`
- [ ] Moderator inicial definido

## Quem entra primeiro

- [ ] Admin entrou e confirmou acesso em `/admin`
- [ ] Moderator entrou e confirmou acesso em `/moderacao`
- [ ] Primeiro grupo de members entrou por magic link
- [ ] Todos os primeiros usuarios concluíram onboarding

## Primeiros testes de uso

- [ ] Um member criou relato de condicoes
- [ ] Um member criou registro economico
- [ ] Um dos fluxos teve anexo valido enviado
- [ ] O detalhe do relato mostra anexo e confirmacoes
- [ ] O detalhe do registro economico mostra anexo e confirmacoes

## Moderacao inicial

- [ ] Moderator revisou os primeiros registros
- [ ] Evento de moderacao foi gravado
- [ ] Relatos ou registros semelhantes foram associados a cluster

## Passagem para pauta

- [ ] Existe pelo menos um cluster com recorrencia real
- [ ] Admin criou uma pauta a partir de cluster
- [ ] Members conseguem apoiar a pauta

## Abertura de nucleo

- [ ] Existe pauta suficientemente clara
- [ ] Existe grupo minimo para tocar essa frente
- [ ] Admin criou nucleo por setor ou tema

## Regras de decisao rapida

### Como um relato vira cluster

- [ ] Ha repeticao do problema
- [ ] Ha peso suficiente no mesmo setor, turno ou unidade
- [ ] Ha urgencia ou prova concreta

### Como um cluster vira pauta

- [ ] O problema ja esta bem nomeado
- [ ] Ja existe base para apoio coletivo
- [ ] O assunto deixou de ser caso isolado

### Quando abrir nucleo

- [ ] A pauta precisa de acompanhamento continuo
- [ ] Ja existe quem vai sustentar a frente
- [ ] O recorte por setor ou tema justifica organizacao dedicada

## Permissoes minimas no piloto

- [ ] `member` so captura, confirma e apoia
- [ ] `moderator` so revisa, associa e registra moderacao
- [ ] `admin` sustenta cadastro, cluster, pauta e nucleo
- [ ] Nenhuma ampliacao de permissao foi feita sem necessidade real
